# Adaptive Graph Reasoning Engine v18.0

## 1. 背景与问题

### 1.1 当前 GraphRAG 的局限

当前系统已经实现：

```
Document
    ↓
Entity Extraction
    ↓
Knowledge Graph
    ↓
Graph Retrieval
    ↓
LLM Answer
```

虽然引入了 Neo4j 和社区图谱，但本质仍然属于：

```
Retrieval-Augmented Generation
```

其主要问题：

### 问题1：图仅用于召回

当前流程：

```
Query
 ↓
Vector Search
 ↓
Entity Match
 ↓
1-Hop Expansion
 ↓
Context
 ↓
LLM
```

图结构只承担：

```
邻居扩展器
```

而非：

```
推理器
```

------

### 问题2：缺乏显式推理路径

当前无法回答：

```
OpenAI CEO曾经领导过哪些组织？
```

因为：

```
OpenAI
 ↓
Sam Altman
 ↓
Y Combinator
```

属于多跳关系链。

系统没有：

```
Path Discovery
```

能力。

------

### 问题3：没有推理可解释性

现有回答：

```
Answer
```

无法展示：

```
为什么得到该答案
```

企业级系统要求：

```
Reasoning Trace
```

# 4. 核心模块设计

------

# 4.1 Query Reasoning Planner

## 目标

把自然语言问题转换为：

```
ReasoningPlan
```

------

例如：

问题：

```
OpenAI CEO曾经在哪些公司工作过？
```

Planner：

```
{
  "query_type":"multi_hop",
  "start_entity":"OpenAI",
  "target_relation":[
    "CEO",
    "WORKED_AT"
  ],
  "max_hops":3
}
```

------

定义：

```
class ReasoningPlan(BaseModel):

    query_type:str

    start_entities:list[str]

    target_entities:list[str]

    target_relations:list[str]

    max_hops:int

    reasoning_strategy:str
```

------

支持：

```
Factoid

Entity Relation

Multi-hop

Comparison

Temporal

Global Summary
```

------

# 4.2 Adaptive Subgraph Retrieval

## 目标

不要把整个Neo4j拉出来。

动态生成：

```
Reasoning Subgraph
```

------

例如：

```
OpenAI
```

只抽取：

```
3-hop内相关实体
```

------

Cypher：

```
MATCH p=(n)-[*1..3]-(m)
WHERE n.name=$entity
RETURN p
LIMIT 1000
```

------

生成：

```
networkx.Graph
```

用于后续推理。

# 4.3 Path Explorer

这是核心创新。

------

输入：

```
ReasoningPlan
```

输出：

```
ReasoningPath
```

------

例如：

```
OpenAI
 ↓
CEO
 ↓
Sam Altman
 ↓
WORKED_AT
 ↓
Y Combinator
```

------

表示：

```
class ReasoningPath(BaseModel):

    nodes:list[str]

    edges:list[str]

    confidence:float

    hop_count:int
```

------

算法：

### Version 1

BFS

```
nx.all_simple_paths()
```

------

### Version 2

Beam Search

```
beam_width=10
```

避免路径爆炸。

------

### Version 3

LLM Guided Search

每轮：

```
当前节点是否值得继续扩展？
```

进行裁剪。

------

# 4.4 Path Ranking

重点创新。

------

不是所有路径都重要。

例如：

```
OpenAI
 ↓
Sam Altman
 ↓
Y Combinator
```

和

```
OpenAI
 ↓
Microsoft
 ↓
GitHub
```

质量不同。

------

定义：

```
score =
w1 * semantic_score
+
w2 * relation_confidence
+
w3 * temporal_consistency
+
w4 * path_length_penalty
```

------

Semantic Score

使用：

```
Embedding(Query)

Embedding(Path Description)
```

计算：

```
cosine similarity
```

------

Relation Confidence

来源：

```
Graph Extraction置信度
```

------

Temporal Consistency

检查：

```
valid_from
valid_to
```

是否冲突。

------

指标：

### Path Recall

```
正确路径是否被召回
```

------

### Path Precision

```
召回路径中有效路径比例
```

------

### Reasoning Accuracy

```
最终答案正确率
```

------

### Faithfulness

```
答案是否由路径支持
```
