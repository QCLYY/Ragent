# 1. Background

## 1.1 Current Problem

Current Ragent AI uses a static retrieval pipeline:

```
User Query
    │
    ▼
Dense Retrieval
    │
Sparse Retrieval
    │
Graph Retrieval
    │
RRF Fusion
    │
Answer
```

Regardless of query type:

- factual question
- multi-hop reasoning
- global summary
- entity relation
- temporal question

all retrieval channels participate equally.

Current assumptions:

```
dense_weight = 0.4
sparse_weight = 0.3
graph_weight = 0.3
```

are globally fixed.

------

## 1.2 Observed Limitation

Graph retrieval is expensive:

| Component        | Latency    |
| ---------------- | ---------- |
| Dense Search     | 50ms       |
| Sparse Search    | 20ms       |
| Neo4j Query      | 200~1000ms |
| Community Search | 100~300ms  |

However:

many queries do not benefit from graph reasoning.

Example:

```
What is Redis?
```

Graph traversal contributes almost nothing.

Yet Neo4j is still queried.

------

Result:

```
Higher latency
Higher token cost
Unnecessary graph expansion
Noise introduction
```

------

## 1.3 Core Hypothesis

Graph retrieval should be used only when it is likely to improve answer quality.

Formally:

```
P(Graph Helpful | Query)
```

should determine retrieval strategy.

Instead of:

```
Always Use Graph
```

we move to:

```
Use Graph When Needed
```

# 4. Query Taxonomy

Introduce six retrieval intents.

------

## Type 1: Factoid

Definition:

single factual answer.

Examples:

```
What is Redis?

What is Kafka?

What is FastAPI?
```

Characteristics:

```
Single entity
Low reasoning
No relationship traversal
```

Strategy:

```
Dense Only
```

------

## Type 2: Entity Relation

Examples:

```
Who founded OpenAI?

Which companies are invested by Tencent?
```

Characteristics:

```
Entity-centric
Relation lookup
```

Strategy:

```
Dense + Local Graph
```

------

## Type 3: Multi-Hop Reasoning

Examples:

```
Which company acquired X and later partnered with Y?
```

Characteristics:

```
Multiple entities
Relation chain
```

Strategy:

```
Graph First
```

------

## Type 4: Global Summary

Examples:

```
Summarize the entire architecture.

What are the major themes of this document?
```

Characteristics:

```
Panoramic understanding
Cross-document aggregation
```

Strategy:

```
Community Summary Retrieval
```

------

## Type 5: Temporal Query

Examples:

```
Who was CEO in 2022?
```

Characteristics:

```
Time-aware
```

Strategy:

```
Temporal Graph Search
```

------

## Type 6: Comparison

Examples:

```
Compare GraphRAG and Vanilla RAG.
```

Strategy:

```
Dense + Sparse + Graph
```
